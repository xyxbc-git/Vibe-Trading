## P0 · 4h 预测引擎（新文件 `jarvis_intraday_predict.py`）

def fetch_kline(symbol: str, interval: str = "4h", total: int = 1500) -> list[dict]:

### T0-2 特征工程 `build_dataset_4h`（纯函数）

- 输入：bars（T0-1 输出）、funding（可选 dict）。全部特征只用 ≤t 的数据：

```python
FEATURE_NAMES_4H = [
  "ret_1", "ret_6", "ret_42",      # 过去 1 根(4h)/6 根(24h)/42 根(7天) 动量
  "vol_30",                          # 30 根已实现波动

  "hour_sin", "hour_cos",           # bar 开盘 UTC 小时周期编码
  "funding",                         # 当期资金费率（缺则 0）
]
```

## P1 · 4h 自动模拟下单（新文件 `jarvis_intraday_trader.py`）

### T1-1 配置组（改 `jarvis_config.py` DEFAULTS）

```python
"intraday": {
    "enabled": True,
    "interval_hours": 4,
    "min_prob": 0.60,            # 开仓最低概率
    "risk_pct_per_trade": 1.0,   # 单笔风险占权益 %（按止损距离反推仓位）
    "max_open_positions": 3,
    "stop_atr_mult": 1.2,
    "take_atr_mult": 1.8,
    "time_stop_bars": 6,         # 6 根 = 24h 未触发即离场
    "cooldown_bars": 1,          # 平仓后隔 1 根才可再开同币
    "max_consecutive_losses": 3, # 连亏 N 次 → 熔断停开仓（见 T2-3）
},
```

- 护栏夹取范围同现有 `_clamp` 模式。
- **验证**：`python jarvis_config.py get intraday` 输出默认组；`_config_smoketest.py` 加 3 条断言。

### T1-2 建表（改 `jarvis_journal.py` `_ensure_db`）

```sql
CREATE TABLE IF NOT EXISTS intraday_positions (
  id INTEGER PRIMARY KEY, symbol TEXT, side TEXT, opened_ts INTEGER,
  entry REAL, stop REAL, take REAL, qty REAL, margin_usdt REAL,
  prob REAL, closed_ts INTEGER, exit_price REAL, exit_reason TEXT,  -- stop/take/time/signal/manual
  pnl_usdt REAL, UNIQUE(symbol, opened_ts));
CREATE TABLE IF NOT EXISTS intraday_predictions (
  id INTEGER PRIMARY KEY, symbol TEXT, bar_ts INTEGER, direction TEXT,
  prob REAL, tradeable INTEGER, entry REAL, stop REAL, take REAL,
  outcome_ret REAL, hit INTEGER,  -- 到期回填：下根实际收益 / 方向是否踩对
  UNIQUE(symbol, bar_ts));
```

- **验证**：删测试库重建，`sqlite3 ... ".schema intraday_positions"` 能看到两表。

### T1-3 `cycle(symbols)` 主流程（每 4h 跑一轮，永不抛出）

1. **盯平仓**：对每个未平仓位取最新价，按 止损→止盈→time_stop_bars→信号反转（最新预测方向与持仓相反且 prob≥min_prob）顺序判定平仓，落 `exit_reason`，记 pnl，推送通知（复用 `jarvis_notify`）。
2. **落预测**：对 watchlist 每币调 `predict_latest`，插 `intraday_predictions`（幂等 UNIQUE）。
3. **回填**：把上一根 bar 的预测用真实收盘回填 outcome_ret/hit。
4. **开仓**：`tradeable and prob>=min_prob and 未熔断 and 持仓数<max_open and 不在冷却` → 按 `risk_pct_per_trade / (stop 距离%)` 反推仓位开模拟仓（账本走 `jarvis_wallet` 同款虚拟权益记账）。
- 方向支持 涨→做多、跌→做空（模拟盘允许空头，字段 side=long/short）。
- **验证**：`python jarvis_intraday_trader.py cycle --symbols BTCUSDT` 手跑一轮，`status` 子命令能列出持仓与累计盈亏。

### T1-4 daemon 接入（改 `jarvis_daemon.py`）

- 新增 `--intraday` 开关：主循环内按「下一个 4h 收盘时刻（UTC 0/4/8/12/16/20 点 + 2 分钟缓冲）」对齐 sleep，唤醒即调 `jarvis_intraday_trader.cycle`，异常隔离不影响原日线心跳。
- 原有 24h record/evaluate 心跳保留（长线视角照旧积累）。
- **验证**：`python jarvis_daemon.py --once --intraday` 单轮通过；日志出现「⚡ 4h 轮」条目。

### T1-5 `_intraday_trader_smoketest.py` + launchd

- 离线 mock predict_latest 与价格，断言：达标开仓/不达标不开、止损平仓、time_stop 平仓、熔断后拒开。
- 更新 `install_launchd` 支持 `--intraday`。
