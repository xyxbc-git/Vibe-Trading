# 贾维斯优化建议 · 信号篇（专家级评审）

> 版本：v1.0（2026-07-19）· 纯只读评审，未改任何代码
> 视角：做市商/大户如何收割散户信号，贾维斯的信号链路哪里会被收割
> 范围：jarvis_twelve_systems（12 套信号 + 共识）、确认层四件套（sentiment/seatbelt/whale_tape/liq_map）、胜率回测（signal_winrate）、反收割组件（stop_hunt/delta_flow）
> 排序：按「对胜率/亏损的影响 × 修复成本」分 P0（必修）/ P1（高价值）/ P2（锦上添花）
> 注：风控篇（做空账目/成交摩擦/仓位封顶等）由另一 agent 负责，本篇只管"信号准不准"，不重复。

---

## 0. 小白术语速查（先读这个）

| 术语 | 大白话 |
| --- | --- |
| 未收盘 bar / 重绘 | 当前这根 K 线还没走完，它的"收盘价"随时在变。用没走完的 K 线算信号，信号会一会儿出现一会儿消失，像画了又擦掉，叫"重绘" |
| 假突破 / 扫流动性 | 价格假装突破关键位，把追突破的人骗进场、把止损单打掉，然后掉头。做市商最常用的收割手法 |
| 扫损（stop hunt） | 大资金故意把价格推到散户集中放止损的位置（前低下方、整数关口），吃掉这些止损单的流动性再反向拉 |
| 磁吸位 | 清算单/止损单密集的价位，像磁铁一样吸引价格插针过去 |
| Delta / CVD / 吸收 | 主动买单减主动卖单叫 Delta，累计起来叫 CVD。价格创新低但 CVD 不再新低=有大资金在下面悄悄接货（吸收），反转概率高 |
| 前视偏差 | 回测时偷看了未来数据，导致统计出的胜率比真实能做到的高 |
| 边沿触发 | 信号从"没有"变成"有"的那一根 K 线才算一次，持续亮灯不重复计数 |
| RR（盈亏比） | 止盈距离 ÷ 止损距离。RR=2 表示赚时赚的是亏时的 2 倍 |

## 1. 总览结论

**这套信号引擎的骨架是专业的**：分层共识（道氏过滤→主线→共振→自适应）、宁缺毋滥的 trade_plan 自洽校验（`_plan` 不自洽返 None）、共识级止损结构锚定+隐蔽化（`stealth_stop_loss` 避开整数关口）、回测同根双触保守计亏、尾部悬空样本丢弃——这些都是散户工具里罕见的正确设计。

**但有三个 P0 级断点会实质性拉低实盘胜率**：

1. **实时信号吃"未收盘 K 线"，回测却用"已收盘 K 线"**——实盘会在假突破的插针瞬间看到（并按）根本不会成立的信号，而回测统计的胜率对应的是另一套更干净的口径。你以为的胜率 ≠ 你拿到的胜率。
2. **确认层四件套只给人看、不给机器用**——自动跟盘开仓完全绕过 sentiment/seatbelt/磁吸位/大单流，四层确认算完就丢。
3. **胜率回测不扣手续费滑点**——15m 级别平均每笔毛利很薄，双边 0.1% 摩擦足以把"正期望"统计变成实盘负期望。

反收割的原材料（扫损检测、Delta 吸收、磁吸位图）**已经全部造出来了，但没接进决策主链**——四条件反转评分（`aggregate_reversal_score`）和 12 系统共识是两个互不通气的平行面板。把已有零件接上，比新造任何指标的性价比都高。

## 2. P0 发现（必修：直接决定胜率统计可信度与实盘成交质量）

### P0-1 实时信号基于未收盘 bar 重绘，与回测口径分裂 —— 假突破追单的根源

**问题**：`fetch_klines_df` 支持 `drop_unclosed=True`（丢弃进行中的 K 线），但两条最关键的消费链都没用它：

- 面板实时信号：`jarvis_dashboard.py` L3646 `/api/twelve/signals` → `jts.fetch_klines_df(sym, iv, 300)`（默认保留未收盘 bar）
- **自动跟盘开仓**：`jarvis_paper_trader.py` L607-612 `_twelve_consensus()` → 同样默认保留

而模块自己在 `jarvis_twelve_systems.py` L26-27 的 docstring 里就承认："输入若含未收盘的最新 bar，信号与点位会随该 bar 实时变化而重绘"。项目里**已经有**正确用法：`jarvis_trend_predict.py` L331、`jarvis_daemon.py` L124、`jarvis_alert_center.py` L716 都传了 `drop_unclosed=True`——唯独信号面板和自动开仓没传。

**为什么降胜率（做市商视角）**：突破系信号（海龟 L170 `close > hh20`、123 法则 L513 `close > tl`、跳空）判定用的"close"在盘中是**最新成交价**。做市商向上插针扫流动性的那 30 秒里，海龟"突破 20 日高"信号满格亮起，daemon 的 `run_cycle` 若恰好在此刻轮询（`jarvis_daemon.py` L211 定时跑 `open_from_twelve`），就在插针顶部市价追多；K 线收盘时针缩回去，信号消失——但仓位已经买在"所有散户都追的那个点"。同时 `record_batch` 会把这些幽灵信号写进变更历史（`twelve_signal_changes`），污染"信号变更"数据。而 `signal_winrate` 回测（L188 `window = df.iloc[i-WINDOW: i+1]` 全部为已收盘 bar）统计出的胜率对应"只在收盘瞬间行动"的纪律——**实盘拿不到这个数**。

**改进方案**：

1. `open_from_twelve` / `_twelve_consensus` 取数改为 `drop_unclosed=True`（一行参数）——自动开仓只认已收盘信号，这是和回测口径对齐的**唯一**途径。
2. 面板路径保留实时性但加标注：`/api/twelve/signals` 响应加 `bar_closed: false` 字段 + 前端在信号格上标"⏳未收盘，可能重绘"；或提供 `?closed_only=1` 参数。
3. 变更历史只记已收盘信号（`record_batch` 调用点判断末根 close_time）。

**预期效果**：消除"插针瞬间追单"这一整类亏损（突破系假信号在 15m/5m 级别占比可观）；回测胜率与实盘胜率口径统一，统计才有意义。
**成本**：S（参数级改动 + 前端一个徽标）。

### P0-2 确认层四件套（情绪/安全带/磁吸位/大单流）只做展示，不进开仓决策

**问题**：dashboard `/api/twelve/consensus`（L3723-3772）辛苦叠加了四层确认：

- `jarvis_sentiment.apply_to_consensus`（L3726-3732）：拥挤度/资金费率背离 → `sentiment.adjusted_confidence`
- `jarvis_seatbelt.apply_to_consensus`（L3736-3742）：Delta 吸收背离三态 → `seatbelt.adjusted_confidence`（conflict 最多 -0.15）
- `jarvis_liq_map.magnet_factor`（L3743-3756）：磁吸位插针警告 → `seatbelt.magnet_warning`
- `jarvis_whale_tape.whale_check`（L3757-3772）：大单净流反向警告 → `seatbelt.whale`

但全局搜索证实：**`adjusted_confidence` 在整个代码库没有任何执行侧消费方**。自动跟盘 `open_from_twelve`（`jarvis_paper_trader.py` L694-706）走的是自己的 `_twelve_consensus()`——直接 `jts.analyze()` + `consensus_multi_tf()`，四层确认**一层都不过**，开仓门槛只看原始 `confidence ≥ 0.45`。磁吸位警告（"现价距强磁吸位仅 0.8%，存在插针风险，谨慎追单"）算出来了，机器照样追单。

**为什么降胜率**：确认层恰恰是这套系统里最"反收割"的部分——技术面看多但 Delta 显示对手方在派发（conflict）、多头拥挤费率过热、现价贴着清算磁吸位——这些正是"技术信号对但入场必被收割"的场景。人看面板会犹豫，机器不会。等于花了四个模块的开发成本，买了零个胜率。

**改进方案**（保持"提醒不阻断"哲学的最小接法）：

1. `open_from_twelve` 增加可选确认门控（配置开关，默认开）：开仓前取 `seatbelt.status`，`conflict` 时跳过本轮（记 skip 原因"Delta 反向背离"）；`magnet_warning.near=True` 且磁吸位在入场方向前方时跳过或缩仓 50%。
2. 置信度换轨：门槛比较对象从原始 `confidence` 换成"叠加后的 `adjusted_confidence`"（sentiment 与 seatbelt 的 delta 相加，夹 [0,1]）——同向共振多开、反向背离少开，平滑版门控。
3. 为避免 dashboard 与 paper_trader 各算一套，把四层叠加逻辑从 dashboard 的 `_calc` 内提炼为 `jarvis_confirm.apply_all(merged, sym)`（新纯函数模块），两侧共用。

**预期效果**：直接砍掉"技术面信号对、微观结构反对"的逆风单——这类单是拥挤行情里亏损的主力；同时磁吸位警告开始真正防插针。
**成本**：M（一个门控函数 + 配置项 + 冒烟；不改确认层本身）。

### P0-3 胜率回测零摩擦：不扣手续费、滑点，SL 按精确价成交

**问题**：`jarvis_signal_winrate.py` `_resolve_sample` L112：`pnl_pct = (exit_price / entry - 1.0) * 100 * sign`——纯价差。入场按触发 bar 收盘价（L189 `close`），离场触 SL 按 SL 价、触 TP 按 TP 价精确成交（L95-107），全程无手续费、无滑点、无点差。

**为什么放大亏损**：币安合约 taker 双边 ≈0.08-0.1%。15m 级别样本（observation 96 根）平均每笔毛盈亏通常在 ±0.5-2% 量级，0.1% 摩擦吃掉 5-20% 的期望值；`payoff_ratio`、`expectancy_pct` 系统性虚高，`low_sample` 之外还有"零摩擦"这个隐藏偏差。更隐蔽的是 SL 成交价：止损位恰在扫损插针路径上，真实成交会**劣于** SL 价（滑点方向永远对你不利），回测按 SL 精确价算，亏损被低估。用户按这份统计选系统、定信任度，会高估短周期系统（摩擦占比大）、低估长周期系统的相对优势。

**改进方案**：

1. `_resolve_sample` 加 `fee_pct`（默认 0.05%/边，配置可调）与 `slip_pct`（SL 成交价额外劣化 0.02-0.05%，或按 ATR 比例）：`pnl_pct -= 2*fee_pct + (触发SL ? slip_pct : 0)`。
2. 结果块并列输出毛/净两套（`expectancy_pct` 与 `expectancy_net_pct`），前端默认展示净值——统计不撒谎。
3. 风控篇已修实盘"成交摩擦"（任务背景注明），回测口径应与实盘摩擦参数**同源**（读同一配置键），避免两边又漂移。

**预期效果**：胜率/期望统计从"实验室数据"变成"到手数据"；短周期系统的真实排名浮出水面，用户选信号的依据变可靠。
**成本**：S（一个函数改动 + 两个配置键 + 冒烟断言）。

## 3. P1 发现（高价值：接上已有零件，胜率增量最大）

### P1-1 反收割四条件评分与 12 系统共识是两个平行世界

**问题**：`jarvis_stop_hunt.aggregate_reversal_score`（L232-347）实现了教科书级的反收割框架——①Delta 背离 ②过程多分布 ③三连分布确认 ④末端止损扫单，4/4=高概率入场。但它只挂在独立 API `/api/reversal-score`（dashboard L4221-4262）；`jarvis_delta_flow` 也只经 seatbelt 进展示链。全局 grep 证实 `jarvis_stop_hunt` 无任何其它消费方——12 系统共识、自动跟盘都不知道它的存在。

**为什么降胜率**："扫损后反转"正是任务背景里点名的做市商行为模式，四条件框架就是为捕捉它设计的。现状是：共识说"看多"，同一时刻反转评分说"刚扫完空头止损、方向看空 3/4"，两个面板各说各话，自动跟盘只听前者。

**改进方案**：把 `aggregate_reversal_score` 作为**择时增强因子**接进共识消费端（不动 12 系统本体）：同向 4/4 → confidence +0.10（相当于"主力刚扫完对手方止损，顺主力方向"）；反向 ≥3/4 → 开仓跳过一轮（正在被反向收割的概率高）。接入位置与 P0-2 的 `jarvis_confirm.apply_all` 合并实现。注意去相关：条件①的 Delta 背离与 seatbelt 是同一数据源（`_delta_payload`），叠加时只计一次（seatbelt 已 confirmed 时四条件的背离分不再额外加成）。
**预期效果**：把"入场时机"从纯指标共振升级为"主力行为确认"，对突破/反转类信号的入场质量提升显著。
**成本**：M。

### P1-2 单系统 trade_plan 的止损没有隐蔽化，胜率统计统计的正是"最容易被扫的摆法"

**问题**：止损隐蔽化（`jarvis_position_calc.stealth_stop_loss`，避开整数关口/摆动点，L125-203）只在**共识聚合层**生效（`jarvis_twelve_systems.py` L1065-1068）。12 个单系统信号自带的 trade_plan 全部是教科书位置：道氏 SL=最近 swing 低点 -0.5×ATR（L232-237）、海龟 SL=entry-2×ATR（L184）、摆动震荡 SL=20 根区间低点 -1×ATR（L683）、缠论 SL=中枢下沿 -0.2×ATR（L448）。其中 0.2-0.5×ATR 的缓冲在 15m 级别就是几十美元，插针常规深度 1×ATR+（stop_hunt 模块自己的 `ATR_MULT=1.0` 就是按"刺破幅度≤1×ATR 算正常扫单"标定的——左手知道插针有多深，右手把止损放在插针范围内）。

**连锁后果**：`signal_winrate` 回测用的正是单系统 plan（L203-207 `sig.get("trade_plan")`）——统计出的"每系统胜率"对应未隐蔽化的止损摆法；而实盘自动跟盘用共识聚合 plan（隐蔽化过）。两套口径，统计对不上实盘。

**改进方案**：

1. `_plan()`（L116-147，所有单系统计划的唯一出口）内统一过一次 `stealth_stop_loss`（entry 已知、方向已知，天然满足其入参）；缓冲下限从 0.3×ATR 提到与 `ATR_MULT=1.0` 对齐的 0.8-1.0×ATR（配置化），RR 随之重算，不自洽照旧返 None。
2. 顺带把 `SL_STRUCT_BUF_ATR = 0.5`（L967）与 stop_hunt 的插针统计对齐复核——0.5 是拍的，stop_hunt 的历史扫单深度分布可以直接算出"90 分位插针深度"作为缓冲依据。

**预期效果**：止损从"扫单区里"挪到"扫单区外"，被针扫掉后行情按原方向走的冤枉止损显著减少（这类是散户最痛的亏损形态）；回测与实盘止损口径统一。
**成本**：S-M（单点改动，但要重跑胜率回测刷新统计）。

### P1-3 摆动系逆势无门控：趋势市里 RSI 超卖接飞刀

**问题**：`signal_oscillator`（L667-701）RSI<30 或 KDJ<20 即出 bullish（strength 0.4-1.0）并附交易计划，无任何趋势状态过滤。`SYSTEM_META` 自己写着"震荡市利器、趋势市毒药：单边行情中超买可以更超买，逆势接飞刀是『买了就跌』重灾区；需道氏顶层过滤配合"（L894-899）——但道氏过滤只在共识层把 layer4 的**层贡献**减半（L1181-1183），单信号本身照常满强度展示+给计划，且边沿触发的回测样本也不区分 regime。单边下跌中 RSI 在 25-35 来回穿越，边沿触发反复产生做多样本，每次都是接飞刀。

**改进方案**：`signal_oscillator` 内加 regime 门控（数据 df 内自足，不引入新依赖）：用现成的 `_swing_points` 判道氏结构或简单 MA50 斜率，趋势与信号反向时 strength 乘 0.4、`trade_plan` 置 None（"逆势超卖只提示、不给计划"）；`reasoning` 注明"逆势信号已降级"。triple_rsi 的逆势金叉同理。
**预期效果**：摆动系的"趋势市毒药"面被拔掉，该系统长/短两侧胜率统计将明显分化改善（可用 winrate 回测直接验证前后差异）。
**成本**：S。

### P1-4 突破系零量能确认：假突破与真突破同权

**问题**：海龟/123/跳空的突破判定纯看价格（L170、L513、L603），成交量列在 df 里但没人看。做市商假突破的经典特征恰是**量能不足的价格越线**（没有真实买盘跟随）。讽刺的是量能尖峰逻辑已在 `jarvis_stop_hunt` 里写好（L99-102 `vol_spike ≥ 1.5×均量`），只是没复用到突破确认。

**改进方案**：突破 bar 量能 ≥1.2×近 20 根均量 → strength 保持；<0.8×均量 → strength 乘 0.5 且 reasoning 标注"缩量突破，假突破风险"；中间线性过渡。只调 strength 不改方向，共识端自然吸收该信息。
**预期效果**：缩量假突破的共识贡献被压低，突破系胜率提升；与 P0-1（已收盘 bar）叠加后，假突破两大特征（盘中插针 + 缩量）都有防护。
**成本**：S。

### P1-5 胜率统计没有反哺共识权重：已有的学习闭环断在最后一米

**问题**：`_LAYER_DEF` 权重（0.30/0.40/0.18/0.12，L953-961）与各系统 strength 公式全是静态经验值；而 `signal_winrate` 已经按 系统×方向×tf 统计出胜率/期望/payoff 并缓存 JSON（L241-244）。数据在手，共识不用——海龟在 4h 上期望 +1.2%、江恩在 15m 上期望 -0.8%，共识里它们的话语权与统计战绩完全无关。

**改进方案**：共识计算时（`consensus()` 入口可选参数）读取 winrate 缓存，对每个系统的 strength 乘一个战绩系数：`clamp(0.5 + expectancy_norm, 0.3, 1.5)`，`low_sample=True` 的系统系数恒 1.0（样本不足不奖惩）。缓存缺失时全部 1.0，零回归。这不是机器学习，只是"谁历史上准就多听谁的"。
**预期效果**：共识从"民主投票"进化为"按战绩加权投票"，长期胜率随样本积累自我改善；坏系统被自动边缘化。
**成本**：M（需防过拟合：系数夹紧 + 低样本豁免 + 按 tf 隔离）。

### P1-6 安全带用 15m 的 Delta 证据确认所有周期的共识

**问题**：`_delta_payload(sym, tf="15m")`（dashboard L3952）缺省 15m，消费点 L3738 `_delta_payload(sym)` 没传 tf——多时间框架共识（主周期通常 4h）拿 15m 的吸收背离做确认。4h 级别反转的吸收行为发生在小时级，15m 窗口（200 根≈2 天）噪声大、`ACTIVE_WITHIN=20` 根≈5 小时就过期，时间尺度错配导致 conflict/confirmed 判定与共识主周期不同步。

**改进方案**：`_delta_payload(sym, tf=merged.get("primary_tf") or "15m")`——按共识主周期取证；或双周期取证（15m+1h）取更严者。
**预期效果**：安全带确认的误报/漏报下降，为 P0-2 接入决策后减少错误门控。
**成本**：S。

## 4. P2 发现（打磨项）

### P2-1 layer4 恒中性成员稀释自适应层

`_LAYER_DEF` layer4 五个成员里 volatility（设计上永远 neutral，L315-340）、martingale（无 trade_history 时恒 neutral，L643-650）、arbitrage（多数时间 neutral）在 `np.mean(contribs)`（L1177）里恒贡献 0——oscillator/gap 真有方向时层得分被除以 5，layer4 实际权重 ≈ 0.12×(2/5)。**建议**：层均值只计"方向性成员"（direction≠neutral 的），或把 volatility 从投票成员改为"波动率调节器"（极端高波动时全局 confidence 打折——它本来就是干这个的）。成本 S。

### P2-2 sentiment 因子族内高度相关，0.75 权重压在同一信息源

`WEIGHTS`（L65-66）里 long_short(0.30)+top_divergence(0.20)+funding(0.25) 都是"持仓拥挤度"的代理变量（同涨同跌），OI(0.25) 半相关——极端行情四因子同时打满，综合分被同一信息放大后触发 `EXTREME_FACTOR_SCORE` 警示与置信度调整。**建议**：拥挤度族内取最强单因子 + 族封顶（如族内合计贡献 ≤0.45），恐贪/链上/爆仓作为独立族。成本 S-M。

### P2-3 止盈没参考磁吸位：TP 摆在"差一点够得着"的位置

`_aggregate_trade_plan_ex` 的 TP 用结构目标中位数/RR 推导价（L1081-1101），不看 `jarvis_liq_map` 的磁吸簇。价格向磁吸位移动后常在簇前减速反转——TP 若落在强磁吸位**外侧**一点点，行情到簇即回头，利润回吐。**建议**：TP1 距强磁吸位（strength≥0.5）±0.3% 内时吸附到簇**内侧**（提前一点落袋）；`magnet_factor` 已有全部数据。成本 S。

### P2-4 自动跟盘共识不传 basis_data，套利系统恒"数据不足"

`_twelve_consensus`（paper_trader L607-612）调 `jts.analyze(df)` 不传 basis——arbitrage 永远 neutral 0.0；dashboard 展示链传了 `_twelve_basis(sym)`。展示与执行的 12 票口径不一致（虽然 arbitrage 本就方向中性、影响极小）。**建议**：跟盘侧也传（或两侧都不传，明确 arbitrage 只做展示）。成本 S。

### P2-5 江恩时间窗的方向判定证据薄弱

时间窗命中后按"现价相对波段中点"给均值回归方向（L367-374，strength 0.4）——时间窗理论只说"变盘概率升高"，方向是拍的。**建议**：降为无方向提示因子（direction=neutral + reasoning 保留变盘提示），或 strength 降至 0.2 以下，避免它在 layer3（权重 0.18，成员仅 2 个）里以 0.4 强度参与方向投票。成本 S。

### P2-6 winrate 回测 stride>1 时边沿判定失真

`last_dir` 每 stride 根才更新（L185-201），stride>1 时中间的翻转被跳过、边沿可能误判。默认 stride=1 无影响。**建议**：文档标注"stride>1 仅供粗扫，正式统计必须 stride=1"，或在结果块回显 stride。成本 S。
