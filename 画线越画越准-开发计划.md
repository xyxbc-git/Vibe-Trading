## 一、问题诊断

现有引擎 `frontend/src/lib/drawings.ts`：

- 5 种线（趋势线 / 支撑压力 / 斐波那契 / 回归通道 / 矩形）全是**纯确定性规则**
- 参数**全部写死**：`SWING_LOOKBACK=5`、`FIB_WINDOW=120`、`CHANNEL_WINDOW=100`、`RECT_WINDOW=40`、S/R 容差 `1.2%`
- 同样的数据永远画出同样的线，**会跟着行情延伸（动态生长），但不会因为「上次画错」而改进（无自学习）**

「越画越准」的本质闭环：

```
画线 → 等未来 K 线验证 → 打分（命中/失效） → 用分数反过来调参/加权
```

## 二、技术方案（四档，本计划落地 A+B+C，D 规划）

### Phase A · 命中率打分地基（公共，所有档位都依赖）

定义「准」= 每种线被未来 K 线验证后的命中率：

| 线型 | 命中定义 | 失效定义 |
| --- | --- | --- |
| 支撑压力 S/R | 价格触及后反弹/被拒 | 直接穿透且不回 |
| 趋势线 | 价格沿线运行、回踩不破 | 有效突破后反向 |
| 斐波那契 | 回调在某 fib 位获支撑 | 所有位都无反应 |
| 回归通道 | 价格在通道内运行（包络命中率） | 频繁冲出通道 |
| 矩形区间 | 突破前停留足够久 | 立刻破坏 |

**实现要点**：用「分割点」把历史 K 线切成两段——前段画线、后段验证。纯函数，可单测、可回测。

### Phase B · 参数自调（推荐先做，立竿见影）

- 把 Phase A 写死的常量改为可传 `DrawParams`
- 加载某币种时拿历史 K 线**网格回测**一组参数（如 `lookback ∈ {3,5,8,13}`、窗口若干档）
- 用 Phase A 的命中率当得分，选该币种+周期下**命中率最高**的一组参数，存起来下次直接用
- 效果：BTC 和小币种不再用同一套参数，每个标的自动适配

### Phase C · 在线可靠度可视化

- 每条线挂生命周期统计：被 touch 几次、反弹几次、破几次 → 实时可靠度
- 可靠度高的线**加粗/高亮**，低的**淡化/隐藏**，label 显示命中率 %
- 效果：画面从「一堆等权线」变成「越靠谱的越突出」

### Phase D · ML / 在线学习（D-1 已落地，D-2/D-3 规划中）

- **D-1（已完成）· 经验先验加权**：`blendReliability` 把当前快照命中率向 `drawingLog` 跨会话累积均值收缩（`w=n/(n+k)`，样本越多越信历史），驱动线条强调度「越用越准」。这是「消费已积累训练集真正从历史学习」的第一步，纯前端、零后端依赖、可单测。
- **D-2（已完成）· 参数 warm-start 记忆**：按 symbol 持久化历史最优参数（`vibe.draw.warm.<symbol>`，不含 bars 数）。缓存未命中时先用 `evaluateParams` 轻量校验历史参数（2 次 walk-forward 评分），仍优于默认即复用、省去全量 1296 次重搜；退化才用 `gridSearchParams({seed})` 全量重搜并播种。seed 作为「保证候选」确保结果单调不劣于历史记忆。直接缓解「缓存键含 bars 数→流式重算」。
- **D-3（已完成）· 结构化特征模型**：`drawingModel.ts` 从 K 线上下文抽 4 维特征（波动率 / 趋势斜率 / 区间位置 / 触碰证据），用在线逻辑回归（BCE+SGD）学习「该上下文下画线是否被尊重（hitRate≥0.5）」。样本带可选 `features` 落进结果日志，组件 `contextModel` 在 ≥15 featured 样本后训练，预测以 0.25 权重门控混入可靠度。后续可选接后端 `jarvis_retrain` 做定期重训 / 加成交量等更丰富特征。
- **样本数据 schema（已存在）**：`DrawingSample{ts,bars,mode,touches,hits,hitRate,baselineHitRate,uplift}`，存储位置 `localStorage: vibe.draw.log.<symbol>`，去重键 `mode:bars`，环形上限 `MAX_SAMPLES=500`。D-2/D-3 直接消费此训练集。

## 三bis、code-audit 收口发现

- **【已修复】neutral 线命中误判**：S/R 横线与斐波那契原判定 `|after-lv|>tol` 会把"价格直接击穿该线"也算命中，但击穿恰是线失效。已改为"价格反弹回原一侧（same-side）才算命中"，击穿不计分。
- **【已大幅缓解】网格搜索同步执行**：`gridSearchParams` 全网格 324 组 × 4 折 ≈1296 次评分曾是冷启动卡顿来源。现已两层缓解：① **P1 坐标下降**——`bars>800` 自动走 coordinate 策略，冷启动评分降到 ≈128 次（约 10×）；② **D-2 warm-start**——流式新增 K 线只做 2 次轻量校验。残余风险仅在「超大数据 + 首次无 warm + 走 full」边角，若仍需进一步可移 Web Worker（需浏览器验证）。
- **【已知限制】缓存键含 bars 数量**：流式新增 K 线会使缓存失效重算；回测历史数据为静态，当前无影响。
- **趋势线方向判定正确**：support `after>lv+tol`、resistance `after<lv-tol` 语义无误，无需改。
