# 15m 短线自动交易系统 — 开发计划

> 版本：v1.0 | 日期：2026-06-24 | 状态：开发中
>
> 基于 QuantDinger Agent Gateway API（方式 B）构建，最大限度减少 QD 页面操作。

---

## 0. 目标

构建一套 **15 分钟级别的全自动短线交易系统**，核心能力：

1. **LLM 驱动策略进化**：自动生成 → 回测 → 分析 → 调整，每轮不重复犯错
2. **不停盘交易**：只要有余额就持续捕捉机会，所有风控参数可配置
3. **通过 QD API 编程调用**：回测、下单、数据全走 API，无需人工操作 QD 页面

## 3. 开发阶段详细设计

### S-01 · 特征工厂（因子菜单）

**状态**：[ ] 未开始

**目标**：定义所有可用于 15m 短线的技术因子，供 LLM 和代码生成器使用。

**因子分类**：

| 分类 | 因子 | 参数范围 |
| --- | --- | --- |
| 趋势 | EMA 交叉、SuperTrend、MACD 柱 | EMA: 5-50, SuperTrend: 7-21 |
| 动量 | RSI 极值、Stochastic、Williams %R | RSI: 6-21, 阈值: 25-35/65-75 |
| 波动率 | 布林带突破、ATR 通道、Keltner | BB: 14-28, ATR: 7-21 |
| 成交量 | OBV 背离、VWAP 回归、量比异常 | 回看: 10-50 |
| 微结构 | 资金费率极值、多空比突变、OI 骤变 | 阈值: 动态百分位 |

**输出格式**（JSON）：

```json
{
  "feature_id": "ema_cross",
  "category": "trend",
  "params": {"fast": 9, "slow": 21},
  "param_ranges": {"fast": [5, 50], "slow": [10, 100]},
  "description": "快慢 EMA 金叉/死叉"
}
```

**验收标准**：
- [ ] 至少 15 个因子，覆盖 5 个分类
- [ ] 每个因子有参数范围定义
- [ ] `list_features()` 返回完整菜单
- [ ] `get_feature("ema_cross")` 返回单个因子详情

### S-02 · 策略代码生成器

**状态**：[ ] 未开始

**目标**：把 LLM 生成的规则 JSON 拼装成 QD 可执行的 Python 策略代码。

**输入**（规则 JSON）：

```json
{
  "name": "supertrend_vol_v3",
  "entry": {
    "conditions": ["supertrend_bullish", "volume_spike"],
    "logic": "AND"
  },
  "exit": {
    "stop_loss_atr_mult": 1.5,
    "take_profit_atr_mult": 2.5,
    "time_stop_bars": 12
  },
  "params": {
    "supertrend_period": 10,
    "supertrend_mult": 3.0,
    "volume_spike_ratio": 1.5
  },
  "direction": "both"
}
```

**输出**：符合 QD `IndicatorStrategy` 规范的 Python 代码字符串。

**关键点**：
- 支持 `open_long / close_long / open_short / close_short` 四路信号
- 止损止盈用 ATR 倍数动态计算
- 生成的代码可直接提交给 QD 回测 API

**验收标准**：
- [ ] 规则 JSON → Python 代码字符串转换正确
- [ ] 生成代码语法无误（`compile()` 检查通过）
- [ ] 支持做多 + 做空双向信号

### S-03 · QD API 回测桥接

**状态**：[ ] 未开始

**目标**：通过 QD Agent Gateway API 提交策略并执行回测。

**调用链**：

```
1. POST /api/agent/v1/strategies  → 上传策略代码
2. POST /api/agent/v1/backtest    → 提交回测任务
3. GET  /api/agent/v1/backtest/{id} → 轮询回测状态
4. GET  /api/agent/v1/backtest/{id}/result → 拿回测结果
```

**回测参数**：

```python
{
    "strategy_id": "...",
    "symbol": "BTCUSDT",
    "timeframe": "15m",
    "start_date": "2025-01-01",
    "end_date": "2026-06-01",
    "initial_capital": 10000,
    "commission_rate": 0.0004,
    "slippage_pct": 0.05
}
```

**验收标准**：
- [ ] 能上传策略代码到 QD
- [ ] 能提交回测并轮询完成
- [ ] 正确解析回测结果（总收益、胜率、最大回撤、盈亏比、Sharpe）
- [ ] 网络异常自动重试

### S-04 · 结果分析器（"验尸官"）

**状态**：[ ] 未开始

**目标**：自动归类亏损原因，生成结构化报告供 LLM 读取。

**亏损原因分类**：

| 原因 | 判定条件 |
| --- | --- |
| 追涨被套 | 入场后 3 根 K 线内即触及止损 |
| 止损太松 | 最大浮盈超过止盈的 50% 但最终亏损 |
| 手续费侵蚀 | 毛收益为正但净收益为负 |
| 方向反复 | 连续 3+ 次同方向亏损 |
| 时间止损浮亏 | 持仓到期时仍为浮亏 |
| 反向轧空 | 做空在上涨趋势中被轧 |

**输出**（结构化 JSON）：

```json
{
  "verdict": "FAIL",
  "total_return_pct": -5.2,
  "win_rate": 42.0,
  "profit_factor": 0.78,
  "max_drawdown_pct": -12.3,
  "sharpe_ratio": -0.5,
  "total_trades": 120,
  "loss_breakdown": {
    "chasing_trapped": {"count": 15, "pct": 35.7},
    "stop_too_loose": {"count": 8, "pct": 19.0},
    "fee_erosion": {"count": 5, "pct": 11.9}
  },
  "improvement_hints": [
    "入场信号需增加趋势确认过滤",
    "止损应收紧至 1.2x ATR"
  ]
}
```

**验收标准**：
- [ ] 正确解析 QD 回测结果中的逐笔交易记录
- [ ] 至少覆盖 6 种亏损原因归类
- [ ] 输出标准 JSON 报告

### S-05 · LLM 进化引擎（核心）

**状态**：[ ] 未开始

**目标**：自动循环「生成策略 → 回测 → 分析 → 调整」，每轮不重复犯错。

**核心数据结构**：

策略墓地 `~/.vibe-trading/scalper_graveyard.json`：

```json
[
  {
    "name": "ema_rsi_v1",
    "rule": {"entry": ["ema_cross", "rsi_extreme"]},
    "result": {"total_return": -5.2, "win_rate": 42, "max_dd": -12},
    "failure_type": "追涨被套",
    "lesson": "单纯 EMA 金叉+RSI 在震荡市频繁假突破",
    "forbidden_combos": ["ema_cross+rsi_extreme 同参数"]
  }
]
```

名人堂 `~/.vibe-trading/scalper_hall_of_fame.json`：

```json
[
  {
    "name": "supertrend_vol_v2",
    "rule": {},
    "result": {"total_return": 8.3, "win_rate": 55, "sharpe": 1.2},
    "oos_validated": true
  }
]
```

**进化循环**：

```
Round 1: LLM 生成规则 → 查重门禁(vs 墓地相似度>80%则重新生成)
         → codegen 拼装代码 → QD API 回测
         → 分析器归类 → 达标？
           是 → 进名人堂 + OOS 验证 → 停止
           否 → 进墓地 + 下一轮
Round 2: LLM 收到上轮分析 + 墓地摘要 → 生成新规则（不得重复）→ ...
...
Round N: 最多 N 轮（默认 10），或找到 ≥1 个达标策略即停
```

**达标条件**（可在 scalper_config.yaml 配置）：

| 指标 | 门槛 |
| --- | --- |
| 胜率 | > 52% |
| 盈亏比 (Profit Factor) | > 1.2 |
| 手续费后净收益 | 正值 |
| 最大回撤 | < 15% |

**LLM Prompt 模板**：

```
你是一个短线策略研究员。基于以下信息生成一个新的 15 分钟策略规则：

## 可用因子菜单
{feature_menu_json}

## 上一轮失败分析
{last_round_analysis}

## 失败策略墓地（禁止重复）
{graveyard_summary}

## 达标策略参考
{hof_summary}

## 要求
- 输出标准 JSON 规则格式
- 不得与墓地中任何策略的因子组合+参数相同
- 必须解释为什么选择这个组合、它为什么不会重蹈覆辙
```

**验收标准**：
- [ ] 能调用 LLM 生成策略规则 JSON
- [ ] 查重门禁正常工作（余弦相似度 > 0.8 触发重新生成）
- [ ] 墓地和名人堂正确读写
- [ ] 完整进化循环跑通（生成 → 回测 → 分析 → 入堂/入墓）
- [ ] `evolve --rounds 3` 能连续跑 3 轮不中断

### S-06 · 15m 自动跟盘引擎

**状态**：[ ] 未开始

**目标**：用进化引擎产出的最优策略，实时 15 分钟自动交易。

**配置文件 `scalper_config.yaml`**：

```yaml
# ===== 风控参数（全部可配置） =====
risk:
  # 单日亏损限额（占总余额比例），默认 warn 只告警不停手
  daily_loss_limit: -0.02
  # 触达限额后的行为：warn=只告警继续交易 / pause=暂停1小时 / stop=停手
  daily_loss_action: "warn"
  # 最大同时持仓数量
  max_concurrent_positions: 3
  # 单笔最大仓位（占总余额比例）
  single_trade_risk: 0.01
  # 低于此余额才真正停止（USDT）
  min_balance_to_trade: 10

# ===== 交易行为 =====
trading:
  # 永不主动停盘，有余额就交易
  always_on: true
  # 信号置信度门槛（0-1）
  confidence_threshold: 0.6
  # 自信模式：信号达标即果断进场
  aggressive_mode: true
  # 连续亏损后冷却K线数（0=不冷却，立即继续）
  cool_down_bars: 0

# ===== 基本参数 =====
timeframe: "15m"
symbol: "BTCUSDT"

# ===== 进化引擎参数 =====
evolve:
  # 最大进化轮数
  max_rounds: 10
  # 达标胜率
  min_win_rate: 52
  # 达标盈亏比
  min_profit_factor: 1.2
  # 达标最大回撤
  max_drawdown_pct: 15
  # 墓地查重相似度阈值
  graveyard_similarity_threshold: 0.8
```

**交易循环**：

```
永续循环:
    拉最新 15m K 线 + 指标数据
    用当前最优策略计算信号
    if 有开仓信号 AND 余额充足 AND 仓位未满:
        果断开仓（不犹豫）
    管理现有持仓:
        检查止盈 / 止损 / 移动止损
    记录战绩到 jarvis_wallet
    回传执行结果到 QD Reflection 服务（触发校准闭环）
    等待下一根 K 线
```

**战绩报表**：

| 指标 | 说明 |
| --- | --- |
| 胜率 | 盈利次数 / 总次数 |
| 盈亏比 | 平均盈利 / 平均亏损 |
| 日均交易次数 | 24h 内成交笔数 |
| 平均持仓时间 | 从开仓到平仓的 K 线数 |
| 累计收益率 | 总盈亏 / 初始余额 |
| 最大回撤 | 峰值到谷底的最大跌幅 |

**验收标准**：
- [ ] 配置文件热加载（修改后不用重启）
- [ ] 永续交易循环正常运行
- [ ] 余额不足时才真正停止
- [ ] daily_loss_action 三种模式都正常
- [ ] 战绩报表输出正确
