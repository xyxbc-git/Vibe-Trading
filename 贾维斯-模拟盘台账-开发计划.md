# 贾维斯 模拟盘台账闭环 — 开发计划

> 版本：v0.1　|　日期：2026-06-17　|　角色：全栈工程师
> 主题：把「模拟盘钱包台账」从「能下单记账」升级为「能看真实盈亏、能自动跟盘、能复盘」的完整闭环。
> 工作方式：**按任务清单逐项推进，完成一项即在第 4 章「进度记录」追加一条。**

---

## 1. 背景与现状盘点

最近已完成（均为未提交新文件）：

- `jarvis_wallet.py`：模拟钱包账本（入金 / 冻结 / 解冻 / 限价挂单簿 / 资金流水）。
- `jarvis_paper_trader.py`：模拟跟盘引擎（按决策开仓 → 盯 → 止损/止盈/到期/反转平仓 + 限价撮合 + `stats()` 盈亏比统计）。
- `jarvis_dashboard.py`：FastAPI 看板已集成台账面板（钱包卡片 / 挂限价单 / 撮合 / 入金 / 挂单簿 / 持仓 / 流水）。
- `_wallet_smoketest.py`：离线全链路冒烟（入金→挂单→撮合→持仓→平仓→对账→撤单→流水），上轮实跑全部通过。

### 已识别缺口

| # | 缺口 | 影响 |
| --- | --- | --- |
| 1 | 持仓表只显示入场价/止损/止盈，**无现价、无浮动盈亏** | 看不到「现在赚还是亏」，台账核心价值缺失 |
| 2 | 看板**没有「自动跟盘」入口**，开仓只能手动挂单 | 无法在 UI 触发引擎按决策自动开/平仓 |
| 3 | **无已平仓历史表、无盈亏比明细** | 复盘只能跑命令行 `report`，看板看不到战绩 |
| 4 | `jarvis_daemon` **未接入** paper_trader cycle | 7×24 引擎不会自动跟盘 |
| 5 | 开/平仓**无自动通知** | 触发动作不推送，需盯屏 |

## 3. 约束与回归

- 全程 **paper-only**，不碰真钱；不改动 `jarvis_wallet` 的资金账本核心算法。
- 每改一处前端/后端，跑 `ReadLints`；涉及钱包/引擎逻辑，跑 `_wallet_smoketest.py` 回归。
- 现价依赖出网（Agent Gateway / brief 因子价）；无网时优雅降级为「—」，不报错。
