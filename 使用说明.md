# 贾维斯 × QuantDinger 使用说明

> 覆盖：研究决策 → 战绩追踪 → 多币雷达 → 执行下单(paper) → 闭环对账 → 通知推送。
> 全程默认 **paper（模拟盘）**，不碰真钱。日期：2026-06-17。

---

## 0. 前置准备（只需一次）

### 0.1 Python 环境
所有 `jarvis_*.py` 都在 `Vibe-Trading/` 下，用项目自带 venv 跑：

```bash
cd Vibe-Trading
./.venv/bin/python jarvis_brief.py BTCUSDT      # 后文命令均可换成此前缀
```

### 0.2 启动 QuantDinger 栈（执行/对账需要）
```bash
cd QuantDinger
docker compose -f docker-compose.ghcr.yml up -d
# 4 容器 healthy 后，浏览器开 http://localhost:8888 （登录 quantdinger / 123456）
```

> 注意：后端 5000 端口直连会 403，**所有 API 都走前端代理 8888**。

### 0.3 签发 Agent Token（执行下单需要，只需一次）
```bash
# 1) 登录拿 JWT
JWT=$(curl -s -X POST http://localhost:8888/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"quantdinger","password":"123456"}' \
  | python3 -c "import sys,json;print(json.load(sys.stdin)['data']['token'])")

# 2) 签发 agent token（R 读 + T 交易，paper-only）
curl -s -X POST http://localhost:8888/api/agent/v1/me/tokens \
  -H "Authorization: Bearer $JWT" -H 'Content-Type: application/json' \
  -d '{"name":"jarvis-executor","scopes":"R,T","paper_only":true,"rate_limit_per_min":120}'
# 返回里的 data.token（qd_agent_...）就是你的执行 token，只显示一次，记下来
```

### 0.4 配置环境变量（执行/对账/通知用）
```bash
export QUANTDINGER_AGENT_TOKEN=qd_agent_你的token
export QUANTDINGER_GATEWAY_BASE=http://localhost:8888   # 默认即此，可省
# 通知（可选）
export JARVIS_TG_BOT_TOKEN=...        # Telegram
export JARVIS_TG_CHAT_ID=...
export JARVIS_FEISHU_WEBHOOK=...      # 飞书机器人 webhook
```

> 也可写进 `~/.vibe-trading/executor_config.json` / `notify_config.json` 持久化，免得每次 export。

## 5. 执行下单 · paper（M1 · 手）

```bash
python jarvis_executor.py BTCUSDT              # 出计划→护栏→下 paper 单
python jarvis_executor.py BTCUSDT --dry-run    # 只跑护栏不真下单
python jarvis_executor.py BTCUSDT --equity 5000  # 覆盖账户权益（用于 sizing）
python jarvis_executor.py --kill-switch        # 急停：撤所有未成交模拟单
python jarvis_executor.py BTCUSDT --json
```

**护栏（自动）**：①只做「偏多（战术）」②信心分≥0.8 ③单笔仓位上限 ④组合风险≤1.5%（超限自动缩仓）⑤必须带硬止损否则拒单 ⑥幂等防重复下单。

下单成功会自动在对账库登记订单号关联（给第 6 步用）。

## 5.5 模拟跟盘引擎（自动买入+卖出+盈亏比）⭐

> 这是你要的「跟盘」：根据决策**自动开仓(买)**，并按 止损/止盈/到期/信号反转
> **自动平仓(卖)**，持续记账，让你观察一段时间的真实盈亏比。

```bash
# 跑一轮：先盯现有持仓该不该平，再找新机会开仓
python jarvis_paper_trader.py cycle --symbols BTC,ETH,SOL

# 常驻跟盘（每 6 小时自动跑一轮，挂后台）
python jarvis_paper_trader.py run --symbols BTC,ETH --interval-hours 6

# 看持仓 + 累计盈亏比（胜率 / profit factor / 浮盈 / 已实现）
python jarvis_paper_trader.py status
python jarvis_paper_trader.py report

# 手动平某币
python jarvis_paper_trader.py close BTCUSDT --reason manual
```

**自动平仓规则**（默认，来自决策计划）：
1. 现价 ≤ 硬止损 → 止损平仓
2. 现价 ≥ 参考止盈 → 止盈平仓
3. 持仓天数 ≥ 时间止损（默认 30 天）→ 到期平仓
4. 最新决策翻成 偏空/中性 → 信号反转平仓

**盈亏比报表**含：胜率、盈亏比(profit factor=总盈利/总亏损)、平均盈/平均亏、
已实现+浮动盈亏合计。持仓/平仓全记在 `~/.vibe-trading/jarvis_journal.db`。

> 建议玩法：`run` 挂常驻跟盘几周 → 定期 `report` 看盈亏比，验证贾维斯的决策到底赚不赚钱。

## 9. 当前已知限制

- **行情取数依赖出网**：若 Docker 后端容器 / 本机当前无法直连 Binance，则
  - executor 下单返回 `fill_price=null`、`status=rejected`（管道正常，只是没成交价）；
  - 对账表显示「待成交/待评估」；雷达对应币会进失败清单。
  - 放开容器/本机出网（或配国内行情源/代理）后，同一命令即返回真实数据，无需改代码。
- 验证容器是否能出网：`docker exec quantdinger-backend curl -m 10 https://api.binance.com/api/v3/time`

## 10. 安全红线

- 默认 **paper-only**；实盘需 token `paper_only=false` + 服务端 `AGENT_LIVE_TRADING_ENABLED=true` 双开关。
- 交易所 API Key 只留本地；agent token 像交易所密钥一样保管，泄漏即撤。
- 急停：`python jarvis_executor.py --kill-switch`。
